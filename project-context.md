### View State Machine (inside ChatWidget)

`activeView`: `'home' | 'chat' | 'help' | 'article'`

Transitions:
- `home ‚Üí chat`: `goToChat()` ‚Äî immediate, `chatPanelOpen = true`
- `chat ‚Üí home`: `goBack()` ‚Äî close ChatPanel animation (200ms), then `activeView='home'`

- `article ‚Üí home`: `goBackFromArticle()` ‚Äî immediate

The 200ms delay matches `CLOSE_DURATION_MS` in both `ChatWidget.tsx` and `ChatPanel.tsx`.

## 6. Component Details

### `ChatWidget` (`ChatWidget.tsx`) ‚Äî 421 lines
The root orchestrator. Key responsibilities:
- **Controlled/uncontrolled open state**: accepts `isOpen` + `onOpenChange` for controlled mode, manages `internalOpen` for uncontrolled
- **Theming**: calls `useTheme(theme)` and `buildCssVars({primaryColor, theme})`, applies result as inline `style` on root `<div>` so all children inherit CSS vars
- **Chat state**: instantiates `useChat({ onMessage, welcomeMessage, initialMessages })`
- **Maximize mode**: `isMaximized` state; when maximized, `TriggerButton` is hidden
- **History on home screen**: if `showHistory=true`, derives `displayRecentMessage` from last session message instead of the `recentMessage` prop
- **Focus management**: `useFocusTrap(panelRef, chatIsOpen)` + returns focus to `triggerRef` when panel closes
- **Keyboard**: `Escape` closes the panel

Props: `onMessage` (required), plus ~18 optional customisation props covering branding, content, layout, and controlled state.

### `ChatPanel` (`ChatPanel.tsx`) ‚Äî 270 lines
The animated sliding panel. Key features:
- **Open/close animation**: two-state machine (`isVisible`, `isClosing`). On close: sets `isClosing=true` ‚Üí CSS exit animation plays ‚Üí after 200ms sets `isVisible=false` (DOM removal). This prevents animation cut-off.
- **forwardRef**: exposes root `div` ref for the focus trap in ChatWidget
- **ARIA**: `role="dialog"`, `aria-modal="true"`, `aria-labelledby={dialogLabelId}`
- **Error banner**: shown when `error` prop is non-null, with a Retry button

Composition: `ChatHeader` + `MessageList` + `InputBar` + optional error banner.

### `ChatHeader` (`ChatHeader.tsx`) ‚Äî 354 lines
Top bar of ChatPanel. Features:
- **Avatar**: 3-way render: URL string ‚Üí `<img>`, ReactNode ‚Üí direct render, undefined ‚Üí auto-initials (up to 2 chars from `agentName`)
- **Status line**: "Online" at rest (green pip + text), animated `TypingIndicator` while `isLoading=true`
- **Buttons**: optional Back (‚Üê), optional Clear (trash), optional Maximize/Restore, required Close (√ó)
- **ARIA**: `<header>` landmark, agent name as `<h2 id={dialogLabelId}>`

Exports two sub-components: `Avatar` and `Logo` (also used by `MessageBubble`).

### `MessageList` (`MessageList.tsx`) ‚Äî 264 lines
Scrollable transcript container. Key features:
- **Auto-scroll**: `useLayoutEffect` scrolls sentinel into view on new messages or streaming chunks
- **Scroll lock**: `IntersectionObserver` watches the sentinel div. When sentinel leaves viewport ‚Üí user scrolled up ‚Üí lock engaged. When sentinel re-enters ‚Üí lock released automatically.
- **Scroll-to-bottom pill**: a floating button appears when `userScrolledUp=true`
- **Empty state**: rendered when `messages.length === 0`
- **ARIA**: `role="log"`, `aria-live="polite"` on the scroll container; messages in a `<ul aria-label="Message history">`

### `MessageBubble` (`MessageBubble.tsx`) ‚Äî 237 lines
Single message renderer. Handles all statuses:
- `loading` ‚Üí `<TypingIndicator variant="bubble" />`
- `streaming` ‚Üí `<MarkdownRenderer isStreaming={true} />` (shows blinking cursor)
- `done` / `undefined` ‚Üí `<MarkdownRenderer />`
- `error` ‚Üí Error icon + message text + Retry button
- User bubbles: right-aligned, `--ai-user-bubble` background
- Assistant bubbles: left-aligned, surface background + avatar (initials or image)
- Timestamp shown on hover/focus, `aria-hidden="true"`

### `InputBar` (`InputBar.tsx`) ‚Äî 280 lines
Message input area. Features:
- **Auto-resize**: `rows={1}` base; on value change, collapses to `height:auto` then sets `min(scrollHeight, MAX_HEIGHT_PX)`. MAX = 5 lines √ó 24px + 18px padding = 138px. Uses `overflow-y: auto` only when capped.
- **Send on Enter**: `Shift+Enter` inserts newline, plain `Enter` calls `onSend()`
- **Autofocus**: `requestAnimationFrame(() => focus())` when `autoFocus=true` to avoid layout jump
- **Re-focus after response**: detects `isLoading` transitioning `true‚Üífalse` and re-focuses textarea
- Disabled state while `isLoading=true`

### `MarkdownRenderer` (`MarkdownRenderer.tsx`) ‚Äî 137 lines
Wraps `react-markdown` with `remark-gfm`. Custom renderers:
- `code`: custom `CodeBlock` with language detection and a Copy button (clipboard API, 2-second "Copied!" feedback)
- `a`: adds `target="_blank" rel="noopener noreferrer"`
- Streaming mode: appends a blinking CSS `<StreamCursor />` span after the markdown

Dependencies: `react-markdown@^10`, `remark-gfm@^4`

### `TriggerButton` (`TriggerButton.tsx`) ‚Äî 186 lines
The floating action button (FAB). Features:
- Circular button anchored to bottom corner by ChatWidget's CSS
- Two icon slots with CSS crossfade: chat bubble/logo (closed) ‚Üî X (open)
- Unread dot: shown when `!isOpen && messageCount > 0`
- Ripple ring animation: plays when `isOpen=true`
- `forwardRef` ‚Äî ChatWidget holds ref for focus return on close
- Accepts all `ButtonHTMLAttributes` as spread (aria-expanded, aria-controls, etc.)

### `WidgetHome` (`WidgetHome.tsx`) ‚Äî 551 lines
The landing screen. Features:
- Dark header with `welcomeMessage` (supports `\n` line breaks) and optional `welcomeSubMessage`
- **Recent message card** (optional): clickable, navigates to chat
- **Status card** (optional): shows `statusText` + `statusUpdated`
- **Help articles**: client-side search with `useMemo` filter on `article.title`; articles either open an `href` in new tab or fire `onArticleClick(article)` for in-widget navigation
- **CTA button**: "Send us a message" ‚Üí `onStartChat()`
- **Bottom nav**: Home (active) | Messages | Help (conditional)

Exports `HelpArticle` interface used across the codebase.

### `WidgetHelp` (`WidgetHelp.tsx`) ‚Äî 294 lines
Full-page help article listing (accessed via Help tab). Same search logic as WidgetHome. Bottom nav with Help tab active. Articles link to `href` or fire `onArticleClick`.

### `WidgetArticle`
Single article detail view. Props: `article`, `onBack`, `onClose`. Not yet visible in public exports (used internally only).

## 7. Hooks Deep-Dive

### `useChat` ‚Äî the core state machine

```
State:
  messages: Message[]      ‚Äî full conversation history
  isLoading: boolean       ‚Äî true from send until response completes
  error: string | null     ‚Äî caught error message

Refs:
  lastUserMessageRef       ‚Äî stores last user text for retryLast()

Flow when sendMessage(text) is called:
  1. Append user Message (status='done') to messages
  2. Append assistant placeholder (status='loading', content='')
  3. Call onMessage(text, historySnapshot)
  4a. If result is ReadableStream ‚Üí handleStream(stream, assistantId)
      - Sets placeholder to status='streaming'
      - Reads chunks via ReadableStream reader + TextDecoder
      - Parses SSE lines: strips 'data: ' prefix, JSON.parse attempts
        to extract .message.content or .content, falls back to raw line
      - Appends to placeholder content via setMessages updater
      - On done: status='done'
      - On error: status='error', content = error message
  4b. If result is string ‚Üí directly populates placeholder (status='done')
  5. setIsLoading(false)

retryLast():
  - Removes last assistant message (the error bubble) from state
  - Removes last user message from state
  - Re-calls send(lastUserMessageRef.current)

clearMessages():
  - Resets to buildInitialMessages() (welcome message or empty)
```

### `useTheme`
- Input: `'light' | 'dark' | 'auto'`
- If not `'auto'`: returns value as-is
- If `'auto'`: reads `window.matchMedia('(prefers-color-scheme: dark)')`, subscribes to `change` events (with legacy `addListener` fallback for old Safari)
- Updates reactively if user switches OS theme mid-session

### `useFocusTrap`
- Activated when `active=true`
- On activation: `requestAnimationFrame(() => firstFocusable.focus())`
- Tab handler: wraps Tab/Shift+Tab within the container's focusable elements
- Focusable selector: `a[href]`, `button:not([disabled])`, `input`, `select`, `textarea`, `[tabindex]:not([tabindex="-1"])`, `details > summary`
- Filters out hidden elements: `el.closest('[hidden]')` or `offsetParent === null`

### `useStreamDetection`
Thin utility: `useMemo(() => value instanceof ReadableStream, [value])`

### `useScrollAnchor`
Thin stub (209 bytes) ‚Äî delegates to MessageList's internal scroll logic.

## 14. Accessibility Design

The SDK is built with WCAG 2.1 compliance as a first-class concern:

- **Dialog**: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to agent name `<h2>`
- **Focus trap**: `useFocusTrap` activates when panel opens; Tab/Shift+Tab cycle inside
- **Focus return**: trigger button gets focus back when panel closes
- **Keyboard**: `Escape` closes, `Enter` sends, `Shift+Enter` newlines
- **Message list**: `role="log"`, `aria-live="polite"`, `aria-relevant="additions"`
- **Streaming bubbles**: `aria-live="polite"` only on in-flight assistant messages
- **Error banner**: `role="alert"`, `aria-live="assertive"`
- **Status line**: `role="status"` with `aria-label="Assistant is online"`
- **Timestamps**: `<time dateTime={ISO}>`, `aria-hidden="true"` (decorative)
- **Avatars and icons**: `aria-hidden="true"` on all decorative SVGs
- **Screen-reader only loading text**: `aria-live="polite"` span in InputBar when loading

## 15. Key Design Patterns

1. **No prop drilling for theming**: CSS variables set once on root `<div>` via inline style; all components inherit via cascade.
2. **Placeholder pattern for streaming**: A `status='loading'` message is inserted immediately; content is appended in-place during streaming; status changes to `'done'` or `'error'`.
3. **Controlled/uncontrolled widget**: `isOpen` + `onOpenChange` for controlled; internal `internalOpen` state for uncontrolled. Pattern mirrors standard form components.
4. **Animation without unmounting**: `isVisible`/`isClosing` two-state in ChatPanel keeps element in DOM during exit animation (200ms), then removes it.
5. **SSR safety**: ChatWidget returns `null` until `mounted=true` (set in `useEffect`). All components marked `'use client'` for Next.js App Router compatibility.
6. **Zero external UI deps**: No icon library, no component library. All icons are inline SVGs. No clsx, no tailwind-merge (custom `cn()` utility).

interface HelpArticle {

## 17. Minimum Consumer Integration

```tsx
import { ChatWidget } from 'ai-chat-interface'
import 'ai-chat-interface/style.css'

function App() {
  const handleMessage = async (message, history) => {
    const res = await fetch('/api/chat', {
      method: 'POST',
      body: JSON.stringify({ message, history }),
    })
    return res.body // ReadableStream for streaming, or res.text() for string
  }

  return <ChatWidget onMessage={handleMessage} />
}
```

The `onMessage` function is the only required prop. Everything else has defaults.
